# EVE作战记录工具 - 部署指南

## 1. 部署概述

EVE作战记录工具是一个纯前端应用，无需后端服务，可通过多种方式部署和使用。本指南将详细介绍部署步骤和注意事项。

## 2. 部署方式

### 2.1 本地部署

**适用场景**：个人使用，无需网络访问

**部署步骤**：
1. **下载代码**：获取完整的项目代码包
2. **解压文件**：将代码包解压到本地目录
3. **启动应用**：双击 `index.html` 文件，在浏览器中打开
4. **验证部署**：确认应用正常加载，功能可正常使用

**优点**：
- 无需网络连接
- 部署简单，即开即用
- 数据存储在本地，隐私性好

**缺点**：
- 无法在多设备间同步数据
- 浏览器缓存清除可能导致数据丢失

### 2.2 本地服务器部署

**适用场景**：团队内部使用，需要在局域网内共享

**部署步骤**：
1. **安装本地服务器**：
   - Windows：使用 XAMPP 或 WAMP
   - macOS：使用 MAMP
   - Linux：使用 Apache 或 Nginx
2. **复制文件**：将项目文件复制到服务器根目录
3. **启动服务器**：启动本地服务器服务
4. **访问应用**：在浏览器中输入 `http://localhost/` 或服务器IP地址

**优点**：
- 局域网内多设备可访问
- 数据存储在本地，安全性高
- 响应速度快

**缺点**：
- 仅限局域网内访问
- 需要配置本地服务器

### 2.3 云服务器部署

**适用场景**：需要公网访问，团队远程协作

**部署步骤**：
1. **选择云服务**：选择合适的云服务提供商（如阿里云、腾讯云、AWS等）
2. **创建服务器**：创建云服务器实例，选择适合的配置
3. **安装Web服务器**：在服务器上安装 Apache 或 Nginx
4. **上传文件**：使用 FTP 或其他方式将项目文件上传到服务器
5. **配置域名**：（可选）配置域名指向服务器
6. **访问应用**：通过域名或服务器IP地址访问应用

**优点**：
- 公网可访问，支持远程协作
- 服务器稳定，数据安全
- 可配置域名，访问方便

**缺点**：
- 需要支付服务器费用
- 配置过程相对复杂
- 依赖网络连接

### 2.4 静态网站托管服务

**适用场景**：快速部署，无需管理服务器

**部署步骤**：
1. **选择托管服务**：选择静态网站托管服务（如 GitHub Pages、Vercel、Netlify等）
2. **创建项目**：在托管服务上创建新项目
3. **上传文件**：将项目文件上传到托管服务
4. **配置设置**：根据服务要求配置项目设置
5. **部署应用**：触发部署流程
6. **访问应用**：通过托管服务提供的URL访问应用

**优点**：
- 部署简单，无需服务器管理
- 大多数服务提供免费额度
- 全球CDN加速，访问速度快

**缺点**：
- 数据存储在浏览器本地，无法在多设备间同步
- 部分服务可能有流量限制

## 5. 数据管理

### 5.1 数据存储
- **存储方式**：使用浏览器 localStorage
- **存储位置**：用户浏览器本地
- **存储限制**：默认5MB（取决于浏览器）

### 5.2 数据备份
- **手动备份**：通过导出JSON功能备份数据
- **自动备份**：建议定期手动导出备份

### 5.3 数据恢复
- **导入备份**：通过导入JSON功能恢复数据
- **注意事项**：导入会覆盖现有数据，请谨慎操作

## 6. 安全考虑

### 6.1 数据安全
- **本地存储**：数据存储在用户本地，不涉及服务器传输
- **隐私保护**：无个人敏感信息收集
- **数据加密**：localStorage存储为明文，建议不要存储敏感信息

### 6.2 访问控制
- **本地部署**：仅限本地访问
- **服务器部署**：可配置访问控制
- **静态托管**：公开访问，注意数据隐私

## 7. 维护与更新

### 7.1 应用更新
1. **下载新版本**：获取最新代码
2. **替换文件**：将旧文件替换为新文件
3. **清除缓存**：清除浏览器缓存，确保加载新文件

### 7.2 常见问题排查

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 页面加载失败 | 文件路径错误 | 检查文件路径是否正确 |
| 功能无响应 | 浏览器兼容性 | 升级浏览器到最新版本 |
| 数据丢失 | 浏览器缓存清除 | 定期导出备份数据 |
| 性能下降 | 历史记录过多 | 清理旧的历史记录 |

### 7.3 日志管理
- **错误日志**：使用浏览器开发者工具查看控制台错误
- **操作日志**：无内置日志系统，建议用户自行记录重要操作

## 9. 技术支持

### 9.1 常见问题

**Q: 数据会丢失吗？**
A: 数据存储在浏览器localStorage中，清除浏览器缓存或更换浏览器会导致数据丢失，建议定期导出备份。

**Q: 可以在多设备间同步数据吗？**
A: 目前不支持自动同步，可通过导出/导入JSON文件手动同步。

**Q: 支持离线使用吗？**
A: 是的，应用是纯前端的，可在离线状态下使用。

**Q: 可以添加新的角色类型吗？**
A: 可以，需要修改相关代码文件。
