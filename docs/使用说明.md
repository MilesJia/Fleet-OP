# EVE作战记录工具 - 使用说明

## 1. 工具简介

EVE作战记录工具是一款专为EVE Online游戏玩家设计的Web应用，用于记录和管理舰队作战全流程，包括集结、回站、战损统计及报告生成。

## 2. 安装与启动

### 2.1 系统要求
- 现代浏览器（Chrome ≥ 90, Firefox ≥ 88, Edge ≥ 90, Safari ≥ 14）
- 互联网连接（首次加载）
- 支持localStorage的浏览器

### 2.2 启动方式
1. **本地启动**：直接双击 `index.html` 文件在浏览器中打开
2. **服务器部署**：将所有文件上传到Web服务器，通过URL访问

## 3. 操作流程

### 3.1 集结开始阶段

1. **填写基本信息**
   - 选择舰队类型（联盟集结、战略行动、本土防御等）
   - 输入集结地点（例如：LW-YEW）
   - 输入FC（舰队指挥）名称
   - 输入预期目标（例如：敌方POS, 收割, 防御）
   - 集结时间会自动生成

2. **选择快捷标签**
   - 点击标签按钮可切换选中状态
   - 支持多选，用于标记作战类型和装备等级
   - 可选标签：PAP、CTA、StratOp、需SRP、无SRP、T1、T2、T3

3. **导入集结成员**
   - 在"导入集结成员"文本框中粘贴成员名单
   - 格式要求：每行一个成员，格式为 `角色名*舰船`
   - 示例：`张三*台风级`
   - 点击"导入集结名单"按钮

4. **编辑成员角色**
   - 导入后会显示成员编辑面板
   - 为每个成员选择合适的角色（火力、后勤、拦截、侦查、电子）
   - 可点击删除按钮移除成员
   - 可点击"添加新成员"手动添加成员
   - 点击"保存修改"按钮保存更改

5. **完成集结阶段**
   - 检查舰队统计数据是否正确
   - 点击"集结完成，进入第二页"按钮

### 3.2 结束回站阶段

1. **填写回站信息**
   - 回站时间会自动生成
   - 输入返回地点（例如：Home Station）
   - 填写作战摘要，描述本次作战的简要情况

2. **导入回站成员**
   - 在"导入回站成员"文本框中粘贴回站成员名单
   - 格式与集结名单一致：`角色名*舰船`
   - 点击"导入回站名单"按钮

3. **登记战损**
   - 系统会自动对比集结和回站名单，显示战损成员
   - 在"战损登记"文本框中粘贴KM链接
   - 格式要求：每行一条KM链接
   - 点击"解析KM并匹配"按钮

4. **完成回站阶段**
   - 检查战损成员是否正确
   - 点击"确认无误，生成报告"按钮

### 3.3 生成报告阶段

1. **查看报告**
   - 系统会自动生成标准化的作战报告
   - 报告包含集结信息、回站信息、成员统计、战损情况等

2. **报告操作**
   - **保存报告**：点击"保存报告"按钮，将报告保存为本地文件
   - **一键复制报告**：点击"一键复制报告"按钮，将报告内容复制到剪贴板
   - **导出JSON**：点击"导出JSON"按钮，将历史记录导出为JSON文件
   - **导入JSON**：点击"导入JSON"按钮，选择之前导出的JSON文件恢复历史记录
   - **历史记录**：点击"历史记录"按钮，查看之前的作战记录

3. **历史记录管理**
   - 在历史记录列表中点击记录可查看详情
   - 点击"清空历史记录"按钮可删除所有历史记录

## 4. 功能详解

### 4.1 成员管理
- **批量导入**：支持从游戏中复制粘贴成员名单，自动解析格式
- **手动添加**：支持单个成员的手动添加
- **角色分配**：为每个成员分配合适的角色，影响统计数据
- **实时统计**：自动统计各角色成员数量及总人数

### 4.2 战损管理
- **自动对比**：通过对比集结和回站名单，自动识别战损成员
- **KM解析**：支持粘贴KM链接，自动解析并匹配战损成员
- **战损统计**：生成详细的战损报告

### 4.3 报告功能
- **标准化报告**：生成格式规范的作战报告
- **多种导出方式**：支持保存、复制、JSON导出
- **历史记录**：自动保存历史作战记录，方便查询和管理

### 4.4 数据管理
- **本地存储**：使用localStorage存储历史记录
- **数据导出**：支持导出为JSON文件备份
- **数据导入**：支持导入JSON文件恢复历史记录
- **数据重置**：支持重置当前填写的数据

## 5. 常见问题

### 5.1 成员导入失败
- **原因**：格式不正确
- **解决方法**：确保每行格式为 `角色名*舰船`，无多余空格

### 5.2 战损匹配错误
- **原因**：回站成员名单格式错误或与集结名单不匹配
- **解决方法**：检查回站成员名单格式，确保与集结名单中的角色名一致

### 5.3 历史记录丢失
- **原因**：浏览器清除缓存或localStorage容量不足
- **解决方法**：定期导出JSON文件备份，避免频繁清除浏览器缓存

### 5.4 KM解析失败
- **原因**：KM链接格式错误或无法访问
- **解决方法**：检查KM链接是否正确，确保网络连接正常

## 6. 最佳实践

1. **提前准备**：在作战开始前打开工具，填写基本信息
2. **实时记录**：作战过程中及时记录重要信息
3. **准确导入**：确保成员名单格式正确，避免错误
4. **及时备份**：定期导出历史记录为JSON文件
5. **规范操作**：按照流程顺序操作，确保数据准确性
